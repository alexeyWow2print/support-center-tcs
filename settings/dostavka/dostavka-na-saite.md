# Доставка на сайте

## Общие положения

Доставка подключается и настраивается в разделе [Интеграции](../integracii/). В данном разделе имеется возможность управлять всеми доступными способами доставки — включать их и выключать.

Перейдя в данный раздел, перед вами откроется список из всех, когда-либо подключенных, способов доставки. В начале списка будут отображаться только активные способы, отсортированные по интеграциям.

![](<../../.gitbook/assets/image (812).png>)

Способ доставки можно включить и выключить с помощью крайнего справа тумблера. Второй тумблер отвечает за доступность доставки при заказе промо продукта.

![](<../../.gitbook/assets/image (800).png>)

Любой продукт на сайте можно отметить как промо продукт, главная особенность подобного продукта — каждый клиент может его заказать лишь один раз (например, пробники)

При клике на способ доставки, откроются настройки этого способа доставки.

![](<../../.gitbook/assets/image (3370).png>)

Среди них имеются:

* **Название**\
  Данный параметр не изменяется.<br>
* **Описание**\
  Данный параметр не изменяется.<br>
* **Фиксированная стоимость**
  * Нет — стоимость будет высчитываться исходя из настроек самой интеграции;
  * Да — появится поле для ввода фиксированного значения стоимости доставки, доставка на сайте всегда будет иметь указанную стоимость.<br>
* **Подпись**\
  Возможность указать текст, который будет отображаться **вместо** цены. Стоимость доставки по-прежнему будет учитываться.<br>
* **Наименование в счете и онлайн-оплате**\
  Возможность указать какое-либо другое название для данного способа доставки, которое будет отображаться во всех документах (акт, товарный чек и т.д.)<br>

## **Если доставка в корзине не загружается**

<figure><img src="../../.gitbook/assets/image (292).png" alt=""><figcaption></figcaption></figure>

При возникновении проблемы с загрузкой доставки в корзине на сайте или в административной панели, следует проверить наличие и актуальность SSL-сертификата вашего сайта.
