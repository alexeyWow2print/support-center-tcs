# Перевод на разные языки

## Общие положения

_В web-to-print платформе TCS имеется возможность перевода контента сайта на разные языки. Здесь мы подробно разберем принцип перевода контента сайта на несколько языков._

## Страницы

Если в [настройках мультиязычности](./#obshie-nastroiki-multiyazychnosti) активировать несколько языков, то в разделе «Страницы» появится возможность добавлять контент на дополнительном языке — кнопка «Добавить».

![](<../../../.gitbook/assets/image (3305).png>)

При наличии модуля мультиязычности, переход в редактор страницы осуществляется путем нажатия на флаг соответствующего языка.\
Первый в списке язык — это язык сайта по умолчанию, все страницы сайта изначально создаются именно на этом языке, остальные языки добавляются вручную.

{% hint style="info" %}
Для каждого языка создается отдельный контент и они никак между собой не пересекаются.
{% endhint %}

### Добавить перевод (страницы)

![](<../../../.gitbook/assets/image (3305).png>)

При нажатии на кнопку «Добавить» появится модальное окно, в котором необходимо выбрать язык, для которого вы хотите добавить перевод.

![](<../../../.gitbook/assets/image (3388).png>)

В списке отображаются те языки, которые выбраны в [настройках мультиязычности](./#obshie-nastroiki-multiyazychnosti).\
При нажатии на «Добавить», добавится соответствующий флаг рядом с языком по умолчанию и откроется пустой конструктор страницы, который необходимо заполнить переводом.

Внутри редактора, в левом верхнем углу, также имеется информация о том какой именно язык сейчас редактируется.\
Здесь же можно переключаться между языками и добавлять новый перевод.

![](<../../../.gitbook/assets/image (866).png>)

{% hint style="danger" %}
Виджеты не имеют настройки переводов.\
Если в виджете имеется контент на конкретном языке, то необходимо создавать копию виджета с переводом на другой язык.\
Например, «Как работает онлайн-заказ (ru)» и «Как работает онлайн-заказ (ua)».
{% endhint %}

## Категория

Перевод категории производится через редактирование категории на вкладке «Перевод»

![](<../../../.gitbook/assets/image (821).png>)

* **Параметр**\
  Первая колонка («Название» и «Короткое описание») включает в себя все параметры категории, которые возможно перевести.<br>
* **Значение**\
  Вторая колонка («Визитки и открытки» и «Не заполнено») — значение параметра категории на языке по умолчанию, то есть которое было указано при создании категории.

{% hint style="info" %}
Если значение параметра не задано — «Не заполнено», то его перевод недоступен.\
Необходимо сначала заполнить данное значение на языке по умолчанию, после этого появится возможность перевода.
{% endhint %}

* **Перевод**\
  Статус перевода на языки — зеленый и красный индикатор сигнализируют о наличии или отсутствии перевода на соответствующий язык.\
  Рядом также имеется иконка карандаша, нажав на который можно заполнить перевод параметра.

При клике на иконку карандаша, откроется следующее модальное окно:

![](<../../../.gitbook/assets/image (3392).png>)

После добавления перевода индикатор перевода поменяется на зеленый и при наведении на него будет отображаться текст перевода:

![](<../../../.gitbook/assets/image (933).png>)

### Продукт

Перевод продукта и его калькуляции производится в настройках продукта на вкладке «Перевод»

![](<../../../.gitbook/assets/image (754).png>)

* **Вкладка продукта**\
  Перевод продукта необходимо осуществить как на вкладке «Общие», так и на вкладке «Калькуляция». Перед параметрами новой вкладки имеется соответствующее название вкладки.<br>
* **Параметр**\
  Первая колонка («Название», «Короткое описание» и т.д.) включает в себя все параметры продукта, которые возможно перевести.<br>
* **Значение**\
  Вторая колонка («Визитка», «Не заполнено» и т.д.) — значение параметра продукта на языке по умолчанию, то есть которое было указано при создании продукта.

{% hint style="info" %}
Если значение параметра не задано — «Не заполнено», то его перевод недоступен.\
Необходимо сначала заполнить данное значение на языке по умолчанию, после этого появится возможность перевода.
{% endhint %}

* **Перевод**\
  Статус перевода на языки — зеленый и красный индикатор сигнализируют о наличии или отсутствии перевода на соответствующий язык.\
  Рядом также имеется иконка карандаша, нажав на который можно заполнить перевод параметра.

При клике на иконку карандаша, откроется следующее модальное окно:

![](<../../../.gitbook/assets/image (3600).png>)

После добавления перевода индикатор перевода поменяется на зеленый и при наведении на него будет отображаться текст перевода:

![](<../../../.gitbook/assets/image (867).png>)

#### Перевод калькуляции продукта

Калькуляция продукта имеет деление на модули. Параметры всех модулей калькуляции, которые возможно перевести, отображаются в настройках продукта на вкладке «Перевод» под соответствующим заголовком.

Каждая деталь имеет свой заголовок и каждый модуль визуально выделен друг от друга.\
На изображении показан модуль «Операция Печать», а именно вкладка « Материал Бумага» и «Количество сторон»:

![](<../../../.gitbook/assets/image (1095).png>)

{% hint style="info" %}
На вкладке «Перевод» отображаются только те модули и параметры калькуляции, которые отображаются на сайте.
{% endhint %}

Если в параметре калькуляции поле «Название варианта параметра» (название, которое отображается для клиента) не заполнено, то на вкладке «Перевод» параметр отображается следующим образом:

![](<../../../.gitbook/assets/image (3372).png>)

Перевод параметра подтягивается непосредственно из справочника и рядом с индикатором перевода имеется ссылка для перехода в справочник конкретного параметра.

## Перевод другого контента

Подобный инструмент перевода имеется во всех разделах сайта.

* **Справочник**:<br>
  * Материал;
  * Операции;
  * Доп. операции;
  * Комплектующие;
  * Товары;
  * Упаковка;
  * Доп. услуги;
  * Цвет;
  * Группировка;
  * Свойства комплектующих.<br>
* **Настройки**:<br>
  * Контактные данные;
  * Настройки оформления заказа;
  * Настройки калькуляции.<br>
* **Другие настройки**:<br>
  * Тэги продукции;
  * Конструктор (шаблоны);
  * Доставка;
  * Меню;
  * Новости / блог;
  * Статусы;
  * Оплата (интеграции);
  * Филиалы.
